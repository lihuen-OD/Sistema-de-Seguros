# UI QA Checklist

Use this checklist before accepting any frontend change.

## Desktop

Check:
- 1366x768
- 1440x900
- 1920x1080

Questions:
- Does the page use the available width properly?
- Does it look like a professional business app?
- Is there excessive empty space?
- Is the content too centered?
- Are cards aligned?
- Are tables full width when appropriate?
- Are filters aligned?
- Are forms grouped correctly?
- Is the topbar aligned with content?
- Is the sidebar consistent?

## Visual bugs

Reject the change if:
- icons overlap numbers
- icons overlap text
- text overflows cards
- numbers overflow cards
- cards have inconsistent heights without reason
- table columns break
- buttons overlap
- modals overflow screen
- page has unwanted horizontal scroll
- layout looks like stretched mobile

## Responsive

Check:
- desktop first
- tablet adaptation
- mobile adaptation
- no broken sidebar on mobile
- no unreadable tables on mobile
- no hidden critical actions

## Product quality

The page must feel:
- complete
- professional
- consistent
- designed
- usable
- enterprise-ready

Reject:
- generic AI layout
- random spacing
- random colors
- too much minimalism
- weak hierarchy
- unbalanced dashboard
- floating cards without structure
